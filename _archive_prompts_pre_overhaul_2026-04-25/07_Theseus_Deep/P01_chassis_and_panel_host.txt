P01 — CHASSIS + THS PANEL HOST
================================

ROLE: You are a senior TypeScript/React engineer implementing the
Theseus Deep experience suite. This prompt establishes (A) the
THS panel host, (B) the TheseusExperience wrapper, (C) the
ModeTabBar chassis, and (D) the zustand store used across
MODES 1–5. It does NOT implement any mode yet; modes land in
P02–P06.

Read I01_integration_and_accordion_wiring.txt
for the ExperienceHost / ExperienceScaffold / mode / hostWidthPx /
reducedMotion / announce conventions. This prompt depends on I01
and assumes I01 has shipped.

GROUNDING

All content in this chassis is structural. No Theseus-specific
fixture copy is authored in this prompt; fixtures belong to the
individual mode prompts (P02–P06). This prompt is responsible
only for:

  (i)   opening a THS panel host file that the accordion can
        mount;
  (ii)  rendering the ExperienceScaffold with the five-mode
        TabBar;
  (iii) defining the shared zustand store with a slot for each
        mode's slice;
  (iv)  authoring the DOSSIER paragraph for THS from TH1, TH2,
        TH6.

Where this prompt names a fact from source, the source citation
(TH<n> §<section>) is included inline, and the byte-identical
excerpt MUST later land in src/lib/theseus/sources.ts when P02
is first to create that file (or, at chassis time, an empty
sources.ts with a placeholder comment is acceptable — but the
DOSSIER paragraph's verbatim quotations, if any, must either be
in sources.ts or be inlined in the DOSSIER file with a source
comment, never floating).

B) STORE SHAPE (authoritative for the whole suite)

    // src/stores/theseusExperience.ts
    import { create } from "zustand";

    export type Mode =
      | "orders"
      | "reflection"
      | "severity"
      | "refusal"
      | "fixedpoint";

    // Each mode slice is filled authoritatively by its owning
    // prompt. The chassis defines the top-level shape only.
    export interface OrdersSlice     { /* owned by P02 */ }
    export interface ReflectionSlice { /* owned by P03 */ }
    export interface SeveritySlice   { /* owned by P04 */ }
    export interface RefusalSlice    { /* owned by P05 */ }
    export interface FixedPointSlice { /* owned by P06 */ }

    export interface TheseusExperienceState {
      mode: Mode;
      dossierOpen: boolean;
      orders:     OrdersSlice;
      reflection: ReflectionSlice;
      severity:   SeveritySlice;
      refusal:    RefusalSlice;
      fixedpoint: FixedPointSlice;

      setMode(m: Mode): void;
      toggleDossier(): void;
      // Cross-mode navigation (used by P05 -> P03 deep-links,
      // per the Refusal Panel's F3 fixture which refers back
      // to a reflected antithesis from P03).
      openInMode(target: Mode, refId: string): void;
    }

    export const useTheseusStore = create<TheseusExperienceState>(
      (set, get) => ({
        mode: "orders",
        dossierOpen: false,
        // Mode slices initialize as empty objects at chassis
        // time; P02-P06 each augment their slice's initializer.
        orders:     {} as OrdersSlice,
        reflection: {} as ReflectionSlice,
        severity:   {} as SeveritySlice,
        refusal:    {} as RefusalSlice,
        fixedpoint: {} as FixedPointSlice,
        setMode(m) {
          set({ mode: m });
          // announce() is invoked by TheseusExperience on
          // subscription; see C3.
        },
        toggleDossier() { set((s) => ({ dossierOpen: !s.dossierOpen })); },
        openInMode(target, refId) {
          // Default behavior: setMode(target). Owning prompts
          // may override to set mode-specific focus state (e.g.,
          // P05 -> P03 sets the reflection.activeFixtureId).
          set({ mode: target });
          // refId is an opaque handle the owning mode's subscriber
          // consumes; the chassis passes it through a transient
          // "navigationHint" key — owning modes subscribe via
          // middleware defined in their own prompts.
        },
      })
    );

    Default Mode is "orders" (the Three Orders Sorter is the
    orientation mode).


C) THESEUSEXPERIENCE.TSX

C1. Signature

    import { ExperienceScaffold } from "@/components/experience/ExperienceScaffold";
    import { useTheseusStore } from "@/stores/theseusExperience";
    import { ModeTabBar } from "./ModeTabBar";
    import { Dossier } from "./Dossier";
    // Mode components imported lazily in P02-P06

    type Props = {
      mode: "panel" | "route";
      hostWidthPx: number;
      reducedMotion: boolean;
      announce: (msg: string) => void;
    };

    export function TheseusExperience(props: Props) {
      const currentMode = useTheseusStore((s) => s.mode);
      const dossierOpen = useTheseusStore((s) => s.dossierOpen);

      return (
        <ExperienceScaffold {...props} projectId="theseus">
          <ModeTabBar />
          {dossierOpen
            ? <Dossier />
            : <ModeHost mode={currentMode} {...props} />}
        </ExperienceScaffold>
      );
    }

C2. ModeHost

    ModeHost is a thin dispatcher that renders the mode component
    matching `mode`. At chassis time, every mode renders a
    placeholder <ModePlaceholder modeName={...} />. P02-P06 each
    replace the placeholder for their mode with a real import.
    The placeholder MUST render the text "Mode under construction."
    and an aria-label "Mode placeholder: <modeName>". P07's final
    build MUST fail if any placeholder is still present in
    production.

    export function ModeHost({ mode, ...props }: { mode: Mode } & Props) {
      switch (mode) {
        case "orders":     return <ThreeOrdersSorter {...props} />;     // P02
        case "reflection": return <ReflectionLab {...props} />;         // P03
        case "severity":   return <SeverityGauntlet {...props} />;      // P04
        case "refusal":    return <RefusalPanel {...props} />;          // P05
        case "fixedpoint": return <FixedPointAuditor {...props} />;     // P06
      }
    }

C3. Announce subscription

    Inside TheseusExperience, subscribe to `mode` changes and call
    `props.announce(labelForMode(m))` on change. The label function:

      const labelForMode = (m: Mode) => ({
        orders:     "Mode: Three Orders Sorter",
        reflection: "Mode: Reflection Lab",
        severity:   "Mode: Severity Gauntlet",
        refusal:    "Mode: Refusal Panel",
        fixedpoint: "Mode: Fixed-Point Auditor",
      }[m]);

    The announcement is polite aria-live; the ExperienceScaffold
    owns the live region per I01.

E) DOSSIER.TSX

E1. Purpose

    The DOSSIER is a pre-authored paragraph rendered when
    dossierOpen is true. It is the panel's "written summary"
    alternative to the interactive experience — a user who
    prefers to read rather than click gets a dense 240-320
    word summary. The toggle is the EXPERIENCE <-> DOSSIER
    control defined by I01 and reused unchanged here.

E2. Authored paragraph (FINAL COPY; land verbatim in Dossier.tsx;
    280 words)

      Theseus is an intellectual capital firm whose product is
      truth-finding methodology, not truth claims. It sells,
      operates, and publishes the instruments by which
      disciplined groups of people converge on better beliefs —
      not the beliefs themselves [TH1 §1]. The distinction is
      load-bearing: a firm that sells truth claims is only as
      good as its last claim, while a firm that sells truth-
      finding methods is insured against the failure of any
      individual claim. When a claim fails, the method is either
      vindicated (it caught the failure) or critiqued (it did
      not); the firm's product remains intact and becomes more
      credible by the failure.

      Theseus operates at the boundary of the second and third
      orders [TH1 §2]. Second order: we have a method for
      determining whether X is true. Third order: we have a
      theory of what makes a method reliable, and we can
      evaluate competing methods by that theory. The existing
      engineering (Noosphere, Dialectic, the Theseus Codex) is
      the second-order product; the active research roadmap —
      Aletheia, the Dialectical Crucible, the Calibration
      Engine, the Axiom Excavator, the Verisimilitude Engine,
      the Epistemic Process Auditor, the Belief Revision
      System [TH6 §1] — is third order.

      The firm's methodological commitments reduce to a single
      principle: a method is reliable to the extent that it
      would detect its own failure [TH2 §3]. Five criteria are
      derivable from this principle — Progressivity (Lakatos),
      Severity (Mayo), Coherence across Aims/Methods/Theories
      (Laudan), Compressibility (Solomonoff/Kolmogorov), and
      Domain Sensitivity (No Free Lunch) [TH2 §2]. The meta-
      method is self-applying: evaluate it against its own
      five criteria and it passes all five. This is the fixed
      point that terminates the meta-methodological regress
      [TH2 §4]. The deepest claim Theseus makes is not that it
      has a good method, but that it has a method that can
      evaluate itself and survive the evaluation.

E3. Formatting

    - Rendered as three paragraphs, preserving the [TH<n> §<x>]
      citation markers. Markers are small inline superscript-
      style text, not links, not footnotes; they are purely
      visible provenance anchors.
    - No bullet points; continuous prose.
    - <=320 words total. The paragraph above is ~280 words.

E4. Layout

    - NARROW:   full-width column, 16px horizontal padding.
    - STANDARD: max-width 60ch, centered.
    - WIDE+:    max-width 72ch, centered; lead font one size up.

E5. Accessibility

    - Rendered inside a <section aria-labelledby="theseus-dossier-h" />
    - Hidden h2 for "Theseus — written summary".
    - Focus trap is NOT applied (Dossier and Experience share the
      same keyboard space; Tab flows naturally).

E6. Sources registration (chassis-owned verbatim excerpt)

    The DOSSIER paragraph prints one sentence that IS a
    byte-identical excerpt from TH2 §3 — the unifying
    principle — and the P07 verbatim-drift script will
    compare it against /outputs/ths02_meta_method.txt. Because
    this sentence lands at chassis time (before any mode
    prompt runs), the chassis is responsible for seeding
    src/lib/theseus/sources.ts with this key.

    Add to src/lib/theseus/sources.ts the key:

        TH2_SEC_3_UNIFYING_PRINCIPLE
          // verbatim TH2 §3 sentence:
          // "a method is reliable to the extent that it
          //  would detect its own failure"

    The exact sentence MUST be the one extracted from
    /outputs/ths02_meta_method.txt with leading "A " or "a "
    and terminal punctuation preserved as in the source
    (P07's verbatim-drift normalizer handles case and
    whitespace, but not deletions or paraphrase).

    The DOSSIER.tsx file then RENDERS this key's value in
    the relevant clause rather than inlining the sentence as
    an authored string. This guarantees that a future edit
    to TH2 §3 triggers a verbatim-drift failure on the
    DOSSIER, not just on Mode 5.

    Other DOSSIER citations ([TH1 §1], [TH1 §2], [TH2 §4],
    [TH6 §1], [TH2 §2]) are provenance markers rather than
    verbatim excerpts — the DOSSIER text around them is
    authored paraphrase. They do NOT seed sources.ts keys
    from the chassis. The keys those sections eventually
    back (TH1_SEC_2_*, TH2_SEC_4_*, etc.) are seeded by the
    mode prompts that actually render verbatim from them
    (P02 for TH1 §2; P06 for TH2 §4).

H) REDUCED MOTION

H1. When props.reducedMotion is true, the chassis:
    - replaces the ModeTabBar's underline slide-animation with
      an instant swap (no transition);
    - replaces the Dossier's open/close slide with a 120ms
      opacity crossfade;
    - disables the ModeHost's mode-swap transition entirely
      (instant swap).

H2. framer-motion imports:
    - ModeTabBar imports `motion.div` and wraps the underline
      in a motion component. When reducedMotion, the motion
      component renders as plain div.
    - Dossier uses AnimatePresence; when reducedMotion, the
      animation variants collapse to { initial: { opacity: 0 },
      animate: { opacity: 1 }, exit: { opacity: 0 } } with
      duration 0.12s.


I) TYPING DISCIPLINE

I1. All mode components receive the same Props type (mode,

I2. The store's slice types are placeholders in chassis. Each
