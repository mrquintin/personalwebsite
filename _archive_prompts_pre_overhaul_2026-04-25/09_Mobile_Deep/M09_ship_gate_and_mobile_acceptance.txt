M09 — SHIP GATE & MOBILE ACCEPTANCE
================================================

PURPOSE
-------
Measurable, mechanical acceptance for the entire 09 suite.
Every item below is a pass/fail check the release is gated
on. Most are automatable (Playwright, Lighthouse, axe-core,
grep); a few are manual (VoiceOver, TalkBack).

When this suite's tasks complete successfully, an agent OR a
human MUST run the checks in this file and record the output.
A failing check blocks merge until resolved.

§A  GREP SHIP-GATES
===================

Run each grep in CI or pre-merge. Any match blocks merge.

A1 — No inline 100vh outside print.css or tokens.css:

     grep -rn '100vh' src/ \
        --exclude='print.css' \
        --exclude='tokens.css'
     # expected: 0 matches (all migrated to 100dvh/100svh per M08 C1)

A2 — No overflow: auto inside .panel-body selectors:

     grep -rn 'panel-body' src/app/globals.css | \
       grep -n 'overflow.*auto'
     # expected: the DESKTOP-only occurrence remains, but no
     # instance is unconditional. Inspect.

A3 — No `window.matchMedia` outside src/components/shell/
     and src/lib/hotkeys/:

     grep -rn 'matchMedia' src/ | \
       grep -v 'src/components/shell/' | \
       grep -v 'src/lib/hotkeys/'
     # expected: 0 matches (M03 §I: mode queries go through
     # useExperienceHost())

A4 — No 100vh inside palette, drawers, or the accordion
     (restated): redundant check against A1, keeping as
     guardrail.

A5 — PRP confinement (carryover from 04/P06 ship-gate):

     grep -rn '.prp-prose\|Source Serif' src/ | \
       grep -v 'src/content/projects/purposeless-efficiency/' | \
       grep -v 'src/styles/prp-prose.css'
     # expected: 0 matches.

A6 — No `unload` event listener:

     grep -rn "addEventListener('unload'" src/
     grep -rn 'addEventListener("unload"' src/
     # expected: 0 matches (M08 F2)

A7 — No third-party carousel / chart library added:

     grep -E 'swiper|keen-slider|splide|chart.js|d3|recharts' \
       package.json
     # expected: 0 matches (M06 §D)

A8 — No runtime-ML, no Three.js on THS:

     grep -rn 'three.js\|three \|onnxruntime\|tensorflow' \
       src/content/projects/theseus/ package.json
     # expected: 0 matches (M07 §F / 07 architectural
     # constraint)

§D  AXE-CORE A11Y
=================

Run axe-core against each of the six URLs above at 320×568
and at 375×667.

Ship-gate: zero axe-core violations of severity "serious" or
"critical". "Moderate" and "minor" violations are logged but
do not block.

Specific checks that must pass:
  [ ] aria-expanded flips correctly on accordion expand.
  [ ] Focus is programmatically moved into an expanded panel
      body (M02 B3).
  [ ] Skip-to-content link is reachable via Tab (Shell.tsx:19)
      and becomes visible on focus-visible (M02 F2).
  [ ] Every interactive element has a touch target ≥44×44 px
      (bot/manual eyeball; axe-core "target-size" rule
      enforces 24×24 by default; elevate to 44×44 via a
      custom rule or manual spec).
  [ ] No keyboard-only interaction without a touch-equivalent
      (M03 §D4, hover-to-expand is desktop-only).
  [ ] Color contrast ≥4.5:1 for normal text, 3:1 for large.
  [ ] All images / SVGs have aria-label or are aria-hidden.

notes, not as an automated gate, but they MUST be performed
before release.

     [ ] Rotor navigation through headings shows panel
         headers.
     [ ] Expanding a panel reads the expand announcement.
     [ ] Inside the expanded body, content is reached in
         logical order.

E2 — TalkBack (Android):
     [ ] Same as E1, substituting TalkBack gestures.

E3 — Dynamic type / font-scale:
     [ ] At iOS "Larger Text" max setting, content remains
         legible with no clipping.
     [ ] At Android font-scale 2.0, content remains legible.

E4 — Low vision / zoom:
     [ ] At browser zoom 400% (desktop), content reflows
         without horizontal overflow (WCAG 1.4.10).
