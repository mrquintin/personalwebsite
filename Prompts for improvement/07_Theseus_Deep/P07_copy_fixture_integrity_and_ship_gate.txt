§C — VERBATIM DRIFT

scripts/theseus/verbatim_drift.ts reads every value in
src/lib/theseus/sources.ts and compares it BYTE-FOR-BYTE
against the matching substring in the corresponding extracted
source text file in /outputs/. The mapping:

    TH1_*   →  /outputs/ths01_orientation.txt
    TH2_*   →  /outputs/ths02_meta_method.txt
    TH3_*   →  /outputs/ths03_research.txt
    TH4_*   →  /outputs/ths04_architecture.txt
    TH5_*   →  /outputs/ths05_pipeline.txt
    TH6_*   →  /outputs/ths06_seven_projects.txt

For each key, the script:
  1. Normalizes both strings: collapses internal whitespace,
     standardizes straight vs. curly quotes (PDF extraction
     commonly produces curly quotes; sources.ts uses
     straight quotes for code-friendliness; the normalizer
     maps between the two one-to-one), and strips trailing
     whitespace per line.
  2. Confirms the key's value appears as a contiguous
     substring of the normalized source file.
  3. Emits a clear diff on failure with line numbers in
     the source file so the drift can be resolved quickly.

§D — BUNDLE BUDGET

scripts/theseus/bundle_budget.ts runs after the production
build and measures the gzipped byte size contributed by the
Theseus panel experience. Per-mode partition:

    Mode 1 (Three Orders Sorter):           ≤ 14KB gzip
    Mode 2 (Reflection Lab):                ≤ 20KB gzip
      — carries the authored SVG projection
        fixtures and the KaTeX-SSR Householder formula.
    Mode 3 (Severity Gauntlet):             ≤ 18KB gzip
    Mode 4 (Refusal Panel):                 ≤ 20KB gzip
      — carries the 9-check pipeline + 6-layer aggregator +
        override modal.
    Mode 5 (Fixed-Point Auditor):           ≤ 14KB gzip
    Chassis (Panel.tsx, TheseusExperience,
             ModeTabBar, Dossier, store,
             typeGuards, sources, shared):  ≤ 8KB gzip
    shipGateAssertions.ts                   ≤ 2KB gzip

    TOTAL                                   ≤ 96KB gzip
    HARD CEILING (I01 budget)               ≤ 80KB gzip

The sum of the per-mode partitions is 96KB, which exceeds
the I01 ceiling. This is INTENTIONAL: it gives each mode a
pessimistic partition while the total still has to clear
80KB. The discipline is:
  - If the total exceeds 80KB gzip, the ship gate fails.
  - If any single mode exceeds its partition, the ship gate
    fails even if the total is under 80KB. The reason:
    violating a per-mode partition signals that mode has
    become unexpectedly heavy (e.g., an animation library
    leaked in), which will compound in later revisions.
  - If the total is under 80KB but a mode has exceeded its
    partition by ≤10%, the ship gate emits a WARNING and
    the implementer may elect to rebalance partitions
    (update the per-mode caps in bundle_budget.ts with
    committed justification) rather than cutting features.
    Rebalances are allowed at most once per quarter; P07's
    git-log check enforces the cadence.

The measurement methodology:
  - Each mode's partition is measured by the size of the
    Webpack/Turbopack chunk that contains its index.tsx
    transitive closure, gzipped with level 9.
  - Shared code (TheseusExperience, the store, shared
    fixtures from sources.ts beyond the mode's direct
    needs) is attributed to the chassis, not to individual
    modes.
  - Third-party dependencies (framer-motion, zustand) are
    attributed to their shared chunk; the Theseus bundle
    measurement is net-of-shared-dep.
  - KaTeX is NOT included at runtime; the Householder
    formula is server-rendered to static HTML at build
    time and embedded as a string. KaTeX's runtime bytes
    must equal zero in the panel measurement; the script
    asserts this explicitly.

§E — CROSS-SUITE ISOLATION

scripts/theseus/cross_suite_isolation.ts enforces the
RELATIONSHIP TO PRIOR SUITES constraints from 00_INDEX. It
is a companion to the prose lint's B3 check but runs at a
wider scope:

  E1. Imports discipline. No file under
      src/components/panels/theseus/ or src/lib/theseus/
      imports anything from src/components/panels/hivemind/
      (CE / Hivemind suites), src/components/panels/
      purposelessEfficiency/, src/lib/purposelessEfficiency/,
      src/lib/hivemind/, or src/lib/coherenceEngine/. The
      Theseus panel stands alone. A cross-import is a FAIL.

  E2. Store discipline. The theseusExperience zustand store
      MUST NOT read from or write to any other suite's
      store. The check walks the store definition and
      its action bodies for any forbidden imports.

  E3. Terminology drift between the six-layer engine (TH4
      §2.3) and the five-layer Coherence Engine (06 suite).
      The script searches for the pattern "five-layer" or
      "5-layer" anywhere in Theseus fixture prose EXCEPT:
        - Fixed-Point Auditor's whyItMatters fields may
          mention "five criteria" freely (the TH2 §3
          reduction).
        - The string "five criteria" is allowed everywhere;
          the string "five-layer engine" / "5-layer
          coherence engine" is forbidden.
      The detection is a regex with a context-sensitive
      exclusion list, tested against the P01-P06 authored
      prose.

  E4. Quintin Hypothesis discipline. The TH3 Quintin
      Hypothesis (geometric detectability of coherence
      predicting outcomes) is a distinct object from the
      CE1 §8 Quintin Hypothesis (pitch-time coherence
      predicting durability). The Theseus fixtures may
      reference the TH3 Quintin Hypothesis by name ONLY
      inside Mode 2 (Reflection Lab) fixtures and ONLY in
      contexts where TH3 §5.2 is the cited source. Any
      other mention is a FAIL.
