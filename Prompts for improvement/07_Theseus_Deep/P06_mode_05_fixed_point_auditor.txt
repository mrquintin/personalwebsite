P06 — MODE 5, FIXED-POINT AUDITOR
===================================

ROLE: You are implementing MODE 5 — the capstone — of the
Theseus Deep experience. The Fixed-Point Auditor is the one
mode the whole suite is arranged to build toward. It answers
the 00_INDEX's Q5: "does the meta-method require a meta-meta-
method, ad infinitum?" The answer the source gives — the
deepest epistemic claim the firm makes — is no: the meta-
method is self-applying, and the regress terminates at a
fixed point. This mode demonstrates that by walking the user
through the self-application, criterion by criterion, each
step rendered with the TH2 §4 source excerpt verbatim and
each step arriving at the same verdict (PASS) that the source
itself arrives at.

The mode is also the single surface that carries the Candid
Frontier — three TH2 §6 anchors rendered verbatim that name
what the meta-method is NOT. These anchors exist to prevent
the mode's triumphal structure (five criteria, five passes,
fixed point) from reading as a closing victory lap. The
Candid Frontier holds the mode's honest posture by literally
surfacing the source's own limits.

Read P01 through P05 first. This prompt
assumes the chassis, all four prior slices, and sources.ts
keys from P01-P05 are in place.

---

GROUNDING

All content in this mode is grounded in:

  TH2 §3     — the unifying principle ("a method is reliable
               to the extent that it would detect its own
               failure") and the five-to-one reduction. The
               mode's opening screen cites this directly.
  TH2 §4     — the fixed-point argument: the self-application
               of the meta-method against its own five
               criteria, producing five PASS verdicts and
               terminating the regress.
  TH2 §6     — "What the meta-method is not": the three
               anchors (certainty; neutrality; NFL
               guarantees the opposite) that hold the
               Candid Frontier sidebar.

TH2 §4 VERBATIM EXCERPTS. Each must be stored byte-identically
in sources.ts. Citation trailer for every TH2 §4 excerpt is
"— II. The Meta-Method, §4".

  TH2_SEC_4_PROGRESSIVE:
    "Progressive? Yes. The meta-method predicts that
    methodologically rigorous firms will outperform
    methodologically sloppy ones on any problem class where
    truth matters, over any long enough horizon. This
    prediction is novel (no prior firm has been explicit
    about it at this level) and testable (the calibration
    scoreboard and external corpus battery test it
    monthly)."

  TH2_SEC_4_SEVERE:
    "Severe? Yes. If methodologically rigorous firms
    consistently underperform on the problem classes the
    firm engages with, the meta-method fails its own
    severity test. The firm has declared the exact
    conditions under which it would abandon the meta-method
    — sustained calibration regression plus sustained
    external-battery failure on matched problem classes."

  TH2_SEC_4_COHERENT:
    "Coherent with aims? Yes. The aim is to identify
    reliable methods. The five criteria (progressivity,
    severity, coherence, compressibility, domain
    sensitivity) are each independently validated by a
    separate intellectual tradition (Lakatos, Mayo, Laudan,
    Solomonoff, NFL). They triangulate."

  TH2_SEC_4_COMPRESSIBLE:
    "Compressible? Yes. Five criteria, each formally
    defined, derivable from a single principle. One slogan
    — 'a method is reliable to the extent that it would
    detect its own failure' — encodes the entire
    specification."

  TH2_SEC_4_DOMAIN_SENSITIVE:
    "Domain-sensitive? Yes. The meta-method explicitly
    acknowledges (via NFL) that no method is universal, and
    its primary output is the mapping from problem class to
    reliable method. The meta-method therefore treats itself
    as subject to the same domain-relativity it imposes on
    its objects."

  TH2_SEC_4_FIXED_POINT_STATEMENT:
    "The meta-method evaluates itself against its own
    criteria and passes all five. This is the fixed point.
    It does not eliminate the abstract possibility of a
    still-deeper meta-meta-method, but it does eliminate
    the necessity of one: the meta-method's authority is
    self-sustaining in the same way that a self-verifying
    proof is, or the way Tarski's undefinability theorem is
    self-referentially consistent when restricted to its
    own metalanguage."

  TH2_SEC_4_DEEPEST_CLAIM:
    "The deepest thing Theseus can claim is not that it has
    a good method, but that it has a method that can
    evaluate itself and survive the evaluation."

TH2 §6 VERBATIM EXCERPTS — the three Candid Frontier anchors.
Each is stored separately, under its own key, because P07's
assertCandidFrontier searches for each first-sentence anchor
individually:

  TH2_SEC_6_NOT_CERTAINTY:
    "The meta-method is not certainty. Nothing in the five
    criteria guarantees that a method applied to a new
    problem will succeed. NFL guarantees the opposite: no
    method succeeds universally. What the meta-method
    guarantees is that when a method fails, the failure is
    visible and informative, and that the firm's stake in
    any single method is appropriately sized to the
    method's historical calibration."

  TH2_SEC_6_NOT_NEUTRALITY:
    "The meta-method is not neutrality. Meta-methodology
    is not 'we have no views.' The firm has strong
    methodological views: that severity beats
    confirmation, that progressivity beats retrofitting,
    that compressibility beats ad-hoc accommodation, that
    domain sensitivity beats universalist overreach. These
    are themselves substantive positions. The meta-method
    is neutral only with respect to first-order claims
    about the world; it is emphatically partisan with
    respect to second-order claims about how belief should
    be produced."

  TH2_SEC_6_NFL_GUARANTEES_OPPOSITE:
    "Nothing in the five criteria guarantees that a method
    applied to a new problem will succeed. NFL guarantees
    the opposite: no method succeeds universally. What the
    meta-method guarantees is that when a method fails,
    the failure is visible and informative, and that the
    firm's stake in any single method is appropriately
    sized to the method's historical calibration."

NOTE on anchor (c3): the TH2 §6 "not certainty" paragraph
contains the sentence used as c3's first-sentence anchor
("Nothing in the five criteria guarantees…"). That sentence
is rendered — and only rendered — inside the
TH2_SEC_6_NFL_GUARANTEES_OPPOSITE block, as a separate card
in the Candid Frontier. The rendered text for c1
(TH2_SEC_6_NOT_CERTAINTY) and for c3
(TH2_SEC_6_NFL_GUARANTEES_OPPOSITE) therefore contain
overlapping prose. THIS IS INTENTIONAL: the 00_INDEX's
assertCandidFrontier searches for each first-sentence anchor
exactly once across all rendered fixture strings, and the
rendered text for c1 and c3 each begins with a distinct
first-sentence anchor. Only the first sentence of each is
the integrity-check target; the overlap after the anchor is
permitted and expected.

The unifying principle (TH2 §3, already stored from earlier
prompts):

  TH2_SEC_3_UNIFYING_PRINCIPLE (already in sources.ts from

§B — STORE SLICE

    type CriterionId =
      | "progressivity"
      | "severity"
      | "coherence"
      | "compressibility"
      | "domain_sensitivity";

    interface FixedPointSliceState {
      cursor: number;               // 0 = opening screen;
                                    //   1..5 = criterion
                                    //   step n; 6 = fixed-
                                    //   point statement;
                                    //   7 = deepest claim
      autoAdvance: boolean;         // default false — this
                                    //   mode is stepped by
                                    //   the user to give
                                    //   the content time
                                    //   to register
      candidFrontierOpen: boolean;  // the sidebar state on
                                    //   NARROW where it is
                                    //   a bottom drawer
      acknowledgedCandid:
        Record<string, boolean>;    // candid anchor ids ->
                                    //   "user has read"
                                    //   flag (used ONLY for
                                    //   aria-live
                                    //   deduplication; no
                                    //   gameplay
                                    //   consequence)
    }

    interface FixedPointSliceActions {
      advanceFixedPoint(): void;    // cursor++ up to 7
      retreatFixedPoint(): void;    // cursor-- down to 0
      resetFixedPoint(): void;      // cursor -> 0
      setFixedPointAutoAdvance(v: boolean): void;
      setCandidFrontierOpen(v: boolean): void;
      acknowledgeCandid(id: string): void;
    }

Initial state:
    { cursor: 0, autoAdvance: false, candidFrontierOpen: true,
      acknowledgedCandid: {} }

The acknowledgedCandid map is NOT surfaced in the UI as a
checklist and does NOT gate any other affordance. Its sole
purpose is to prevent the aria-live channel from re-
announcing an anchor if the user returns to Mode 5 and the
same anchor is re-focused within the same session. This is
a deliberate UX choice: the anchors are substantive content,
not achievements; no progress bar, no "X of 3 read" meter,
no completion badge.

    cursor = 0  ── advanceFixedPoint ──►  cursor = 1
    cursor = n  ── advanceFixedPoint ──►  cursor = n+1
                                          (up to 7)
    cursor = n  ── retreatFixedPoint ──►  cursor = n-1

§H — reducedMotion

  - Step transitions (cursor change) are opacity-only
    crossfades over 1 frame at reducedMotion; at normal
    motion they are 200ms crossfades with a subtle 4px
    translate.
  - No auto-advance at any motion setting: this mode is
    always stepped by the user, to give the content time
    to register. (autoAdvance flag exists in the slice
    but its default is false; if set to true it still
    forces a minimum 2000ms per step, dropping to 0ms at
    reducedMotion in which case the step changes are
    manual regardless.)
  - The CandidFrontier cards do not animate on open; the
    drawer expands with a height swap at normal motion,
    instant at reducedMotion.

    export function assertCriterionStep(
      o: unknown
    ): asserts o is CriterionStep {
      invariant(typeof o === "object" && o !== null, ...);
      const s = o as Record<string, unknown>;
      invariant(
        ["progressivity","severity","coherence",
         "compressibility","domain_sensitivity"]
          .includes(s.id as string),
        "CriterionStep.id must be one of the 5 CriterionId " +
        "literals");
      invariant(
        [1,2,3,4,5].includes(s.ordinal as number),
        "CriterionStep.ordinal must be 1..5");
      invariant(typeof s.criterionName === "string"
        && (s.criterionName as string).length > 0,
        "criterionName required");
      invariant(typeof s.sourceExcerpt === "string"
        && (s.sourceExcerpt as string).length > 0,
        "sourceExcerpt required (verbatim TH2 §4)");
      invariant(typeof s.sourceKey === "string"
        && (s.sourceKey as string).startsWith("TH2_SEC_4_"),
        "sourceKey must be a TH2 §4 key");
      invariant(s.verdict === "PASS",
        "Every criterion-on-self verdict is PASS; TH2 §4 " +
        "enumerates five PASS verdicts and this is the " +
        "fixed-point claim");
      // Only the severity step carries a non-null
      // falsificationNote; the source declares the
      // abandonment condition in-line for severity only.
      if (s.id === "severity") {
        invariant(typeof s.falsificationNote === "string"
          && (s.falsificationNote as string).length > 0,
          "Severity step MUST carry a non-null " +
          "falsificationNote");
      } else {
        invariant(s.falsificationNote === null,
          "Non-severity steps must have null " +
          "falsificationNote");
      }
    }

    export function assertCriteriaOnSelfArray(
      arr: unknown
    ): asserts arr is readonly CriterionStep[] {
      invariant(Array.isArray(arr), "must be array");
      invariant(arr.length === 5,
        "Fixed-Point Auditor requires exactly 5 " +
        "CriterionStep objects; TH2 §4 enumerates five " +
        "criteria.");
      const ordinals = (arr as any[]).map(s => s.ordinal);
      invariant(
        JSON.stringify(ordinals) === JSON.stringify([1,2,3,4,5]),
        "CriterionStep ordinals must be [1,2,3,4,5] in " +
        "order");
      const ids = new Set((arr as any[]).map(s => s.id));
      invariant(ids.size === 5
        && ids.has("progressivity")
        && ids.has("severity")
        && ids.has("coherence")
        && ids.has("compressibility")
        && ids.has("domain_sensitivity"),
        "All five CriterionId literals must be present " +
        "exactly once");
      for (const s of arr) assertCriterionStep(s);
    }

    export function assertCandidAnchor(
      o: unknown
    ): asserts o is CandidAnchor {
      invariant(typeof o === "object" && o !== null, ...);
      const a = o as Record<string, unknown>;
      invariant(
        ["not_certainty","not_neutrality","nfl_opposite"]
          .includes(a.id as string),
        "CandidAnchor.id must be one of the 3 literals");
      invariant([1,2,3].includes(a.ordinal as number),
        "CandidAnchor.ordinal must be 1..3");
      invariant(typeof a.firstSentenceAnchor === "string"
        && (a.firstSentenceAnchor as string).length > 0,
        "firstSentenceAnchor required");
      invariant(typeof a.fullExcerpt === "string"
        && (a.fullExcerpt as string).startsWith(
          a.firstSentenceAnchor as string),
        "fullExcerpt must begin with the " +
        "firstSentenceAnchor (P07 verbatim-drift " +
        "invariant).");
      invariant(typeof a.sourceKey === "string"
        && (a.sourceKey as string).startsWith("TH2_SEC_6_"),
        "sourceKey must be a TH2 §6 key");
      invariant(typeof a.whyItMatters === "string"
        && (a.whyItMatters as string).length > 0,
        "whyItMatters required");
    }

    export function assertCandidFrontierArray(
      arr: unknown
    ): asserts arr is readonly CandidAnchor[] {
      invariant(Array.isArray(arr), "must be array");

      invariant(uniq.has(
        "Nothing in the five criteria guarantees that a " +
        "method applied to a new problem will succeed."),
        "Anchor c3 first-sentence text required verbatim");

generic "meta-epistemology" language. The prose in
CriterionStep's framingQuestion fields and the

probabilistic, geometry, information, judge — TH4 §2.3) may

mode's prose must keep the six-layer engine clearly named
and must not use the string "five layers" except where
quoting TH2 §3's "five criteria" reduction.
