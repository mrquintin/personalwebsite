P04 — MODE 3, SEVERITY GAUNTLET
=================================

ROLE: You are implementing MODE 3 of the Theseus Deep experience.
The Severity Gauntlet is the panel in which the user directly
practices Mayo's severity criterion: given a claim and a proposed
test, the user decides — alternative by alternative — whether
the test would have caught the claim's falsity under each
plausible alternative hypothesis. The canonical verdict is
authored (severe / non-severe / partially severe) and is revealed
with a source-grounded rationale and, where applicable, a pointer
to the production analog in Noosphere.

This mode exists because severity is the single criterion whose
violation produces the most common methodological failure —
confirmation bias. The 00_INDEX declares this mode's teaching
target as Q2 (the five formal properties and their reduction to a
single principle), specifically the severity half of that
question. Mode 2 (Reflection Lab) showed HOW severity is made
generative (by reflecting the thesis to produce an antithesis);
Mode 3 shows WHAT severity is in its raw form (a property of the
test-claim pair, assessed against plausible alternatives).

Read P01, P02, and P03 first. This prompt assumes
the chassis is in place, OrdersSlice and ReflectionSlice are
functional, and sources.ts / typeGuards.ts already carry the
keys enumerated in P01–P03.

---

GROUNDING

All content in this mode is grounded in:

  TH2 §2.2   — Severity (Mayo). The statement, the provenance
               (Error and the Growth of Experimental Knowledge;
               Statistical Inference as Severe Testing), the
               operational test, the production analog
               (adversarial.py), and the load-bearing reason
               ("severity inverts the criterion: you must show
               that your method would have rejected the claim
               under plausible alternatives").
  TH2 §3     — the unifying principle and severity's role within
               it ("severity is the criterion applied to a single
               claim within a single test").
  TH2 §5     — engineering consequences ("severity ⇒ adversarial
               challenge + counterfactual replay").
  TH4 §2.4   — geometry.py / meta_analysis.py; counterfactual
               replay via the CorpusSlicer (severity made
               computational — eliminates temporal leakage).
  TH5 §8     — Stage 8 of the epistemic pipeline: the adversarial
               challenge subsystem, its four challenge states
               (pending / addressed / fallen / fatal), and the
               declared targets ("attack the severity," "attack
               the empirical grounding," "attack the
               domain-transfer assumption").
  TH6 §3     — the Dialectical Crucible (severity made generative;
               the Crucible does not wait for an adversary, it
               creates the strongest one structurally).

The Mayo severity statement, to be rendered VERBATIM in the
mode's footer and stored byte-identically in sources.ts under
the key TH2_SEC_2_2_SEVERITY_STATEMENT:

    "A method provides severe evidence for a claim H only if
    the method would very probably have detected H's falsity,
    had H been false. A test that would pass any reasonable
    hypothesis in the same neighborhood is non-severe."
                                                   (TH2 §2.2)

The Mayo severity operational test, to be rendered verbatim in
the SeverityVerdict sidebar and stored under
TH2_SEC_2_2_SEVERITY_OP_TEST:

    "For any claim the firm endorses, it asks: was this
    analysis conducted in a way that would have detected the
    claim's falsity under plausible alternative scenarios? Or
    was the analysis structured such that any nearby hypothesis
    would have been confirmed by the same evidence?"
                                                   (TH2 §2.2)

No other TH2 §2.2 content is rendered in this mode. The
"severity is the computational formalization of steelmanning"
sentence is a helpful summary but is NOT shown in the UI; the
UI shows the definitional statements only. The reasoning for
this restriction is that the mode is about application, not
about summary slogans; the slogans risk moving the user toward
vibes-based assessment rather than the alternative-by-
alternative test the mode demands.

§B — STORE SLICE

Augment src/stores/theseusExperience.ts with a SeveritySlice. The
shape:

    type AlternativeMark =
      | "unset"
      | "distinguished"
      | "not_distinguished";

    interface SeveritySliceState {
      activeFixtureId: string | null;    // null = picker view
      // keyed first by fixtureId, then by alternativeId
      marks: Record<string, Record<string, AlternativeMark>>;
      // true iff user has asked to reveal the verdict for
      // the active fixture at least once this session
      revealed: Record<string, boolean>;
    }

    interface SeveritySliceActions {
      pickFixture(fixtureId: string): void;
      markAlternative(
        fixtureId: string,
        alternativeId: string,
        mark: AlternativeMark
      ): void;
      revealVerdict(fixtureId: string): void;
      resetFixture(fixtureId: string): void;
      // clears everything in the slice; used by the "reset
      // mode" affordance in P07
      resetAllSeverity(): void;
    }

Initial state:

    {
      activeFixtureId: null,
      marks: {},     // every fixture's marks map seeded to
                     //   { [altId]: "unset" } lazily on
                     //   pickFixture
      revealed: {},
    }

openInMode(target, refId) from P01 is overridden in the
severity slice: when target === "severity", refId (when set)
MUST be interpreted as a fixtureId and the store MUST set
activeFixtureId to refId before returning. (This is how Mode 4
refuses a PublicationCandidate whose adversarial challenge
points back at a Severity fixture.)

State selectors exported from the same module:

    selectActiveSeverityFixture(state): SeverityFixture | null
    selectSeverityMarks(fixtureId): Record<altId, AlternativeMark>
    selectSeverityCompletion(fixtureId):
        { markedCount: number; totalCount: number;
          allMarked: boolean }
    selectSeverityRevealed(fixtureId): boolean

No Severity action mutates any other slice. revealVerdict is the
only action that announces via the reducedMotion-aware announce
channel (see §H).

F1 — SEVERE (investment domain)
  id: "f1_moat_net_retention"
  domainLabel: "investment thesis"
  claim:
    "Company X has a durable competitive moat attributable to
    network effects, not merely to switching costs or the
    current credit regime."
  proposedTest:
    "Measure Company X's net revenue retention across three
    distinct regulatory and macroeconomic regimes: pre-GDPR
    (2017), post-GDPR (2019), and the 2022 credit tightening.
    Require NRR ≥ 110% in all three regimes. Additionally
    require that in at least one of the three regimes, a
    direct competitor's NRR fell by ≥ 5 percentage points
    while Company X's NRR held. Pre-register the three
    regimes, the 110% threshold, the 5pp competitor-decline
    clause, and the competitor set before examining the data."
  alternatives:
    [
      { id: "f1_alt_switching_costs",
        statement:
          "Company X's retention is explained by switching
          costs, not network effects. Customers stay because
          leaving is expensive, not because the product
          becomes more valuable as more customers join.",
        canonicalDistinguished: true,
        rationale:
          "Network effects predict strengthening retention
          when the installed base grows AND a competitor's
          retention collapses (because network value is
          concentrated in the winning platform). Pure
          switching-cost moats do not require competitor
          decline. The competitor-decline clause therefore
          discriminates network effects from switching-cost
          lock-in." },
      { id: "f1_alt_credit_regime",
        statement:
          "Retention is high because the current credit regime
          has kept capital cheap and corporate IT budgets
          unusually generous; in a tightening, NRR would
          collapse.",
        canonicalDistinguished: true,
        rationale:
          "The three-regime requirement explicitly includes a
          credit tightening (2022). If NRR held in that
          regime, the alternative is falsified. If NRR did
          not hold in that regime, the claim is falsified.
          Either way the test discriminates." },
      { id: "f1_alt_lagging_cohorts",
        statement:
          "Retention is a lagging indicator. It reflects deals
          closed three to five years ago; Company X may
          already be losing new cohorts but the averaged NRR
          does not yet show it.",
        canonicalDistinguished: true,
        rationale:
          "The test requires three CONSECUTIVE years of NRR ≥
          110%, which is only attainable if renewed cohorts
          (not just legacy ones) expand. A rolling three-year
          NRR above threshold forces renewal by newer
          cohorts; legacy-only expansion mathematically
          cannot sustain it." },
      { id: "f1_alt_cyclical_industry_tailwind",
        statement:
          "Retention is elevated for the whole industry in
          this period — every comparable company also has NRR
          ≥ 110%. There is nothing distinctive about Company
          X.",
        canonicalDistinguished: true,
        rationale:
          "The competitor-decline clause fails in the
          scenario where the entire industry's NRR holds. The
          test therefore returns a negative verdict in the
          case where nothing distinctive is happening, which
          is the correct behavior." }
    ]
  canonicalVerdict: "severe"
  verdictRationale:
    "The test distinguishes the network-effects claim from
    the three most serious alternatives (switching costs,
    credit regime, lagging cohorts) and correctly refuses to
    confirm in the null case where the industry broadly
    retains well. The pre-registration clause eliminates the
    post-hoc rescue — having declared the 110% threshold and
    the competitor-decline clause before examining the data,
    the analyst cannot relax the criterion after seeing it.
    This is severity in Mayo's sense: the test would probably
    have detected the claim's falsity under each of the
    plausible alternatives, had the claim been false."
  productionPointer:
    "noosphere/noosphere/adversarial.py generates the
    structured 'attack the severity' challenge against a
    firm-tier investment thesis; counterfactual replay via
    CorpusSlicer (evaluation/) enforces the pre-registration
    discipline by pinning the store to the pre-test as_of
    date when the adversarial run executes (TH4 §2.4)."
  sourceAnchor: "TH2_SEC_2_2_SEVERITY_STATEMENT"
  domainTransferNote: null

F2 — SEVERE (methodology domain; the engine-on-itself claim)
  id: "f2_engine_reliability"
  domainLabel: "methodology"
  claim:
    "The six-layer coherence engine is reliable on ideological
    texts in the sense that, on paragraphs where three
    independent coders agree on the coherence label, the
    engine agrees with the coders' label above its
    pre-registered threshold and abstains (UNRESOLVED) within
    its pre-registered band."
  proposedTest:
    "Construct a held-out test set of 500 ideological
    paragraphs, drawn from sources disjoint from any text
    that entered the engine's training or tuning corpus
    (provenance audit required). Label each paragraph
    independently by three coders as COHERENT / INCOHERENT /
    MIXED; retain only paragraphs with 3-of-3 agreement as
    the consensus set, and retain the remainder as the
    contested set. Pre-register (a) the precision and recall
    thresholds, (b) the tolerated UNRESOLVED band, and (c)
    the failure-mode taxonomy (four named failure classes).
    Apply the engine. Pass only if precision and recall both
    exceed the pre-registered thresholds on the consensus
    set AND the UNRESOLVED rate lies within the pre-
    registered band on the full set AND no observed failure
    falls outside the declared taxonomy."
  alternatives:
    [
      { id: "f2_alt_training_leakage",
        statement:
          "The engine appears to work because the test set
          happened to resemble its training distribution —
          near-duplicates leaked through the corpus-audit
          process.",
        canonicalDistinguished: true,
        rationale:
          "The provenance-audit requirement forces a
          cryptographic-hash comparison of every test
          paragraph against the engine's training-corpus
          manifest. Near-duplicates are flagged by
          embedding-similarity screening above a
          pre-registered threshold and excluded." },
      { id: "f2_alt_unresolved_loophole",
        statement:
          "The engine's apparent 'reliability' is an artifact
          of a generous UNRESOLVED setting: on any hard
          case, the engine returns UNRESOLVED, and so its
          precision on cases it commits to is high simply
          because it never commits on the hard ones.",
        canonicalDistinguished: true,
        rationale:
          "The pre-registered UNRESOLVED band prevents
          widening the abstention zone to mask errors. If
          the engine abstains more often than the band
          permits, the test fails even if precision on the
          committed-to cases is high." },
      { id: "f2_alt_coder_unreliability",
        statement:
          "The coders' labels are themselves unreliable; an
          'agreement' between the engine and the coders may
          be two random-ish processes agreeing by chance.",
        canonicalDistinguished: true,
        rationale:
          "The 3-of-3 agreement restriction for the
          consensus set is exactly the inter-coder
          reliability gate. Cohen's kappa must clear a
          pre-registered floor. The contested set is
          handled as an unresolved-rate check, not as a
          label source." },
      { id: "f2_alt_post_hoc_failure_rescue",
        statement:
          "When a failure occurs, the team labels it as a
          'known failure mode' and excludes it from the
          denominator.",
        canonicalDistinguished: true,
        rationale:
          "The pre-registered four-class failure taxonomy
          closes this hatch: any failure outside the
          taxonomy is counted as an unhandled failure, and
          a single unhandled failure trips the
          pre-registered stop condition." }
    ]
  canonicalVerdict: "severe"
  verdictRationale:
    "Every loophole that would allow the test to confirm
    the claim under a plausible alternative has been closed
    by the pre-registration clauses: training-leakage via
    hash audit, UNRESOLVED gaming via the pre-registered
    band, coder noise via the 3-of-3 consensus cut, and
    post-hoc failure relabeling via the declared taxonomy.
    This is methodological severity in Mayo's strong sense.
    It also exemplifies the firm's own stance toward its
    own engine: the engine is evaluated by a test the
    engine could fail."
  productionPointer:
    "The analog is noosphere/noosphere/meta_analysis.py
    (the five-criterion gate applied to candidate
    principles) composed with the external_battery/ corpus
    framework (TH2 §2.5; TH5 §8). Counterfactual replay

    as_of date so the thresholds cannot be adjusted ex
    post."
  sourceAnchor: "TH2_SEC_2_2_SEVERITY_STATEMENT"
  domainTransferNote: null

F3 — NON-SEVERE (confirmation-bias trap)
  id: "f3_aggregate_brier"
  domainLabel: "calibration"
  claim:
    "Theseus's public forecasting track record is well-
    calibrated."
  proposedTest:
    "Compute the firm's overall Brier score across every
    public prediction resolved to date. If the aggregate
    Brier falls below a named threshold, declare the firm
    well-calibrated."
  alternatives:
    [
      { id: "f3_alt_difficulty_mix",
        statement:
          "The predictions resolved so far happen to have
          been drawn from low-difficulty problem classes
          (base-rate-heavy, short horizon). The aggregate
          Brier reflects question selection, not skill.",
        canonicalDistinguished: false,
        rationale:
          "Aggregate Brier is not stratified by difficulty
          class or horizon. A low score on easy questions
          is indistinguishable from a low score on hard
          questions when only the aggregate is reported." },
      { id: "f3_alt_selective_publication",
        statement:
          "The firm selectively publishes predictions it is
          confident in and privately retains uncertain ones.
          The aggregate therefore measures selection, not
          underlying calibration.",
        canonicalDistinguished: false,
        rationale:
          "The test examines only the published-and-resolved
          set. It is silent on the denominator. No
          selective-publication hypothesis is ruled out." },
      { id: "f3_alt_resolution_favoritism",
        statement:
          "Ambiguous outcomes have been resolved in the
          firm's favor when interpretation is discretionary;
          the aggregate reflects resolution decisions, not
          predictive accuracy.",
        canonicalDistinguished: false,
        rationale:
          "Aggregate Brier does not audit resolution
          decisions. No resolution-favoritism hypothesis is
          ruled out by this test structure." },
      { id: "f3_alt_domain_masking",
        statement:
          "Good calibration on some domains masks bad
          calibration on others. The firm may be excellent
          on economics and poor on technology; the average
          tells neither story.",
        canonicalDistinguished: false,
        rationale:
          "An aggregate is a single scalar. By construction
          it cannot reveal domain-conditional failures. A
          Brier decomposition (calibration, resolution,
          uncertainty) stratified by domain would reveal
          this; the proposed test does none of that." },
      { id: "f3_alt_brier_asymmetry",
        statement:
          "Brier score itself is vulnerable to asymmetric
          event frequencies: on mostly-false-prediction
          classes, a constant low-probability forecast
          attains a deceptively good score.",
        canonicalDistinguished: false,
        rationale:
          "The proposed test does not compute a base-rate
          decomposition or a skill score relative to a
          climatological baseline. The aggregate does not
          discriminate against the trivial low-probability
          forecaster." }
    ]
  canonicalVerdict: "non-severe"
  verdictRationale:
    "This is the canonical confirmation-bias trap. Mayo's
    own warning: 'A test that would pass any reasonable
    hypothesis in the same neighborhood is non-severe.'
    An aggregate Brier score below a threshold would be
    produced by a genuinely well-calibrated firm, by a
    selectively-publishing firm, by a resolution-favoring
    firm, by a domain-masking firm, and by a
    base-rate-exploiting firm. The test cannot discriminate
    between these hypotheses. It produces a confirming
    number for every plausible neighbor of the claim, which
    is the exact structural failure severity is defined
    against. The remediation is not to raise the threshold;
    it is to replace the test with a severe one: Brier
    decomposed into calibration, resolution, and
    uncertainty components, stratified by method and
    domain, with an audit of publication selection and
    resolution decisions. That severe replacement is the
    Calibration Engine (TH6 §4), not the single-number
    report."
  productionPointer:
    "The severe replacement exists in partial form at
    noosphere/noosphere/scoring.py (Brier decomposition)
    and in the external_battery/ corpus framework
    (external audit). TH6 §4 describes the full Calibration
    Engine, which generalizes the scoreboard with the
    Method Attribution cross-reference. The Epistemic
    Process Auditor (TH6 §7) is the system that would flag
    THIS non-severe test as a Laudan coherence failure
    between the firm's stated aim (calibrated forecasting)
    and the method actually used to evaluate it (aggregate
    Brier)."
  sourceAnchor: "TH2_SEC_2_2_SEVERITY_STATEMENT"
  domainTransferNote: null

F4 — PARTIALLY SEVERE (NFL hook — domain-sensitive)
  id: "f4_reflection_coherence_by_domain"
  domainLabel: "ideological text"
  claim:
    "The Householder reflection v' = v − 2(v·â)â, applied to
    ideologically-charged source sentences across a named
    concept axis, produces structurally coherent antitheses
    that a blinded reviewer would classify as arguments
    rather than as grammatical malformations."
  proposedTest:
    "On domain A (ideological text with a named concept axis
    — CLASS, LIBERTY, EQUALITY, EFFICIENCY): select 40 source
    sentences across the CLASS axis. Apply the Householder
    reflection; retrieve the nearest five real-world
    sentences from the authored reference corpus; render the
    reflected argument. Three blinded reviewers rate the
    reflected output on coherence (1-5). Pass the domain-A
    test if mean coherence ≥ 4.0 and inter-reviewer kappa ≥
    0.6. Domain-B extension: re-apply the same test to 40
    empirical claims about causal mechanisms in climate
    attribution, reflected across a candidate axis
    'naturalistic / anthropogenic'."
  alternatives:
    [
      { id: "f4_alt_reviewer_priming",
        statement:
          "Reviewers are primed by the juxtaposition of
          original and reflected sentences; they rate the
          reflected sentence as coherent because the
          contrast structure is suggestive, not because the
          reflected sentence stands alone as an argument.",
        canonicalDistinguished: true,
        rationale:
          "Blinded review means reviewers see the reflected
          sentence without the original or the axis label.
          Priming by contrast is therefore excluded." },
      { id: "f4_alt_corpus_nearest_neighbor_work",
        statement:
          "The reflected sentence's coherence comes from the
          nearest-neighbor lookup in the authored reference
          corpus, not from the reflection operation itself.
          Any random vector's nearest neighbor from a
          well-curated corpus will be coherent.",
        canonicalDistinguished: false,
        rationale:
          "The proposed test does not ablate the
          nearest-neighbor step. A severe version would
          include a control arm in which the reflection is
          replaced by a random vector of equal norm, with
          the same nearest-neighbor lookup applied. Without
          that control arm, the test cannot attribute
          coherence to the reflection rather than to the
          corpus." },
      { id: "f4_alt_axis_selection",
        statement:
          "The technique works on paradigmatic ideological
          axes (class, liberty, equality) because centuries
          of political discourse have produced
          well-populated antipodes on those axes; it will
          not work on axes without that discourse history.",
        canonicalDistinguished: false,
        rationale:
          "The domain-A test is restricted to the CLASS
          axis. It cannot speak to axes with sparser
          discourse histories. The domain-B extension
          attempts to test one such axis (naturalistic /
          anthropogenic) but the proposed test applies the
          same coherence-rating protocol without adjusting
          for the fact that empirical claims do not have
          'antipodes' in the discursive sense ideological
          claims do." }
    ]
  canonicalVerdict: "partially-severe"
  verdictRationale:
    "On domain A, the test is severe against the
    reviewer-priming alternative but NON-severe against the
    nearest-neighbor-does-the-work alternative, because no
    control arm isolates the reflection's contribution. It
    is also silent on the axis-selection alternative, which
    is a coarser domain-sensitivity issue. The domain-A
    result therefore cannot carry more than one cheer: the
    method appears to produce coherent antitheses on the
    CLASS axis, but the authored test cannot attribute the
    coherence to the reflection operator itself. On domain
    B, the test is INAPPLICABLE. Empirical claims about
    causal mechanisms do not have a 'concept axis of
    contention' in the sense ideological claims do;
    reflecting them across a 'naturalistic / anthropogenic'
    axis produces something grammatical that is not a
    counter-hypothesis, because the claims do not have
    structural antipodes. Extending the domain-A test to
    domain B without modification would be a domain-
    sensitivity failure in the No Free Lunch sense
    (TH2 §2.5). This fixture therefore introduces the NFL
    hook handled in Mode 5 (Fixed-Point Auditor): a test's
    severity is domain-relative, and a meta-method that
    does not track that relativity is itself non-severe."
  productionPointer:
    "noosphere/noosphere/adversarial.py handles the
    reflection-based challenge on domain A. The external
    corpus battery (external_battery/) tracks per-corpus
    failure modes; the 'domain-transfer' attack target
    (TH5 §8) is the production analog of the severity
    failure shown here when the reflection is naively
    applied across a domain it was not validated on. See
    TH3 §5.3 (Domain-Relative Coherence) for the firm's
    published position on this boundary."
  sourceAnchor: "TH2_SEC_2_2_SEVERITY_STATEMENT"
  domainTransferNote:
    "On domain B (empirical causal claims) the authored
    test is inapplicable: the axis 'naturalistic /
    anthropogenic' is not a concept axis of contention in
    the sense the reflection operator requires; applying
    the test without modification would produce
    grammatically well-formed but structurally empty
    outputs and would count as a domain-sensitivity
    violation (NFL). A different severity apparatus is
    required in domain B, which Mode 5 develops as the
    domain-to-method mapping primary output of the
    meta-method."

§G — reducedMotion

When reducedMotion === true (from I01):
  - verdict reveal is a single opacity crossfade from 0 to
    1 over 1 frame (effectively instantaneous); no scale
    or translate animations on the verdict tile.
  - segmented-control toggles change state with NO
    transition animation (immediate swap).
  - the footer severity-statement expand-collapse at NARROW
    shows and hides immediately with no height animation.

  - verdict tile fades in over 200ms and settles with a
    subtle 1px inset-shadow pulse (one cycle); respects

    transition.

after minification, per the per-mode budget partition
declared in P07).

    export function assertSeverityFixture(
      f: unknown
    ): asserts f is SeverityFixture {
      invariant(typeof f === "object" && f !== null,
        "SeverityFixture must be an object");
      const g = f as Record<string, unknown>;
      invariant(typeof g.id === "string" && g.id.length > 0,
        "SeverityFixture.id must be a non-empty string");
      invariant(typeof g.domainLabel === "string",
        "SeverityFixture.domainLabel must be a string");
      invariant(typeof g.claim === "string"
        && g.claim.length > 0,
        "SeverityFixture.claim must be non-empty");
      invariant(typeof g.proposedTest === "string"
        && g.proposedTest.length > 0,
        "SeverityFixture.proposedTest must be non-empty");
      invariant(Array.isArray(g.alternatives)
        && g.alternatives.length >= 3
        && g.alternatives.length <= 5,
        "SeverityFixture must have between 3 and 5 " +
        "alternatives (MODE 3 spec in 00_INDEX).");
      const altIds = new Set<string>();
      for (const alt of g.alternatives as unknown[]) {
        invariant(typeof alt === "object" && alt !== null,
          "alternative must be an object");
        const a = alt as Record<string, unknown>;
        invariant(typeof a.id === "string" && a.id.length > 0,
          "alternative.id must be non-empty");
        invariant(!altIds.has(a.id as string),
          "alternative ids must be unique within a " +
          "fixture");
        altIds.add(a.id as string);
        invariant(typeof a.statement === "string"
          && (a.statement as string).length > 0,
          "alternative.statement must be non-empty");
        invariant(typeof a.canonicalDistinguished === "boolean",
          "alternative.canonicalDistinguished must be a " +
          "boolean");
        invariant(typeof a.rationale === "string"
          && (a.rationale as string).length > 0,
          "alternative.rationale must be non-empty");
      }
      invariant(
        g.canonicalVerdict === "severe"
        || g.canonicalVerdict === "non-severe"
        || g.canonicalVerdict === "partially-severe",
        "canonicalVerdict must be one of the three " +
        "literals");
      invariant(typeof g.verdictRationale === "string"
        && (g.verdictRationale as string).length > 0,
        "verdictRationale must be non-empty");
      invariant(typeof g.productionPointer === "string"
        && (g.productionPointer as string).length > 0,
        "productionPointer must be non-empty");
      invariant(typeof g.sourceAnchor === "string"
        && ((g.sourceAnchor as string)
            .startsWith("TH2_SEC_2_2_")
            || (g.sourceAnchor as string)
            .startsWith("TH5_SEC_8_")),
        "sourceAnchor must be a TH2 §2.2 or TH5 §8 key");
      invariant(
        g.domainTransferNote === null
        || (typeof g.domainTransferNote === "string"
            && (g.domainTransferNote as string).length > 0),
        "domainTransferNote must be null or a non-empty " +
        "string");
    }

    export function assertSeverityFixtureArray(
      arr: unknown
    ): asserts arr is readonly SeverityFixture[] {
      invariant(Array.isArray(arr),
        "Severity fixture array must be an array");
      invariant(arr.length === 4,
        "00_INDEX MODE 3 spec requires exactly 4 severity " +
        "fixtures (two severe, one non-severe, one " +
        "partially severe).");
      const severe = arr.filter(
        (f: any) => f.canonicalVerdict === "severe"
      ).length;
      const nonSevere = arr.filter(
        (f: any) => f.canonicalVerdict === "non-severe"
      ).length;
      const partiallySevere = arr.filter(
        (f: any) => f.canonicalVerdict === "partially-severe"
      ).length;
      invariant(severe === 2,
        "Expected exactly 2 severe fixtures; found " +
        String(severe));
      invariant(nonSevere === 1,
        "Expected exactly 1 non-severe fixture; found " +
        String(nonSevere));
      invariant(partiallySevere === 1,
        "Expected exactly 1 partially-severe fixture; found " +
        String(partiallySevere));
      const partial = arr.find(
        (f: any) => f.canonicalVerdict === "partially-severe"
      );
      invariant(
        partial && typeof (partial as any).domainTransferNote
          === "string",
        "The partially-severe fixture must carry a " +
        "non-null domainTransferNote; MODE 3 NFL-hook " +
        "requirement.");
      const ids = new Set<string>(
        arr.map((f: any) => f.id)
      );
      invariant(ids.size === 4,
        "fixture ids must be unique");
      invariant(ids.has("f1_moat_net_retention")
        && ids.has("f2_engine_reliability")
        && ids.has("f3_aggregate_brier")
        && ids.has("f4_reflection_coherence_by_domain"),
        "Severity fixture ids are load-bearing; P05 " +
        "PublicationCandidate F3 references " +
        "f1_moat_net_retention.f1_alt_switching_costs.");
      for (const f of arr) assertSeverityFixture(f);
    }
