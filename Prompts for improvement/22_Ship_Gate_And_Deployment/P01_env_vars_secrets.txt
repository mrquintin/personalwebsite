22/P01 — ENVIRONMENT VARIABLES AND SECRETS
=============================================

GOAL
----
A single source-of-truth doc for what env vars exist, where they
come from, what each does, and how the local dev setup loads them.

§A FILES TOUCHED
----------------
  .env.example                           (rewrite)
  src/lib/env.ts                         (NEW — typed env access)
  src/lib/env.test.ts                    (NEW)
  docs/v2_env.md                         (NEW)

§B ACCEPTANCE
-------------
B1.  .env.example lists every variable consumed by the app, grouped:

       # === Public site config ===
       NEXT_PUBLIC_SITE_URL=http://localhost:3000

       # === LLM (Anthropic) ===
       ANTHROPIC_API_KEY=sk-ant-...

       # === Embeddings ===
       VOYAGE_API_KEY=pa-...
       EMBEDDER=voyage      # or "openai"
       OPENAI_API_KEY=sk-... # only if EMBEDDER=openai

       # === Vector store (Vercel Postgres) ===
       POSTGRES_URL=postgres://...
       POSTGRES_URL_NON_POOLING=postgres://...

       # === Rate limit (Vercel KV / Upstash) ===
       KV_REST_API_URL=https://...
       KV_REST_API_TOKEN=...

       # === LLM tuning ===
       LLM_MODEL=claude-opus-4-7
       LLM_RATE_LIMIT_HOUR=20
       LLM_RATE_LIMIT_DAY=100

B2.  src/lib/env.ts:
       - validates env on module load (zod schema)
       - exposes a typed `env` object: env.ANTHROPIC_API_KEY, etc.
       - throws a clear error at startup if a required var is
         missing, naming the variable
       - distinguishes server-only from client-safe (NEXT_PUBLIC_)

B3.  Server-only vars (ANTHROPIC_API_KEY, etc.) MUST NOT leak to
     the client bundle. The validator throws if accessed in a client
     context. Verify by attempting to import env.ANTHROPIC_API_KEY
     in a "use client" file — TypeScript / runtime should refuse.

B4.  docs/v2_env.md walks the user through:
       - Anthropic Console: where to create API key
       - Voyage Console: where to create API key
       - Vercel Postgres: how to create the DB and link to project
       - Vercel KV (Upstash Redis): same
       - How env vars sync to local .env.local via `vercel env pull`
       - How to rotate a secret (immediate revoke + new key + redeploy)

B5.  .env.example is committed. .env.local is gitignored.

§C VERIFICATION COMMANDS
------------------------
  test -f .env.example
  test -f src/lib/env.ts
  test -f docs/v2_env.md
  npx vitest run src/lib/env.test.ts

§D NON-GOALS
------------
  • Do NOT commit any real secret.
  • Do NOT use process.env directly outside src/lib/env.ts.
