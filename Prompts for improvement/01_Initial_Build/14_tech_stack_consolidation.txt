DECISION TREE (execute in this order)
-------------------------------------
1. Make the Next.js app the canonical site.
   - Every new page lives under src/app/.
   - The shell described in Prompt 02 wraps everything.
   - Tokens from Prompt 01 are in src/styles/tokens.css and loaded by
     src/app/layout.tsx.

2. Demote the legacy /public site to ARCHIVE status.
   - Move public/index.html → public/_legacy/index.html so it no longer
     resolves at the root (if you care to keep it reachable for
     reference; otherwise delete it — git history preserves it).
   - Move public/pages/, public/blog/, public/projects/, public/css/,
     public/js/ all under public/_legacy/ or delete outright.
   - EXCEPTION: public/js/ontology-data.js and
     public/pages/ontology.html contain data that will be ported to
     /theseus (Prompt 09). Do NOT delete them until the port is done.
     Copy their data into src/content/projects/theseus/ first, then
     archive.
   - Audit tools/publish.py and tools/config.json. If they are bound
     to the legacy site, archive. If they support the Next.js
     workflow (unlikely), keep.

3. Install canonical dependencies.
   Add as dependencies (current stable as of May 2025; agent should

4. Decide on Tailwind vs CSS modules.
   RECOMMENDATION: adopt Tailwind, but configured as a thin mapping
   over the tokens from Prompt 01. Reasons:
     - The visual system is token-heavy and Tailwind's arbitrary
       values and theme extension make token consumption terse.
     - Agents (Claude Code, Cursor) produce faster, more consistent
       Tailwind output than CSS modules.
     - Utility-first co-locates layout with markup, which suits
       dense terminal layouts.
   If rejected, CSS modules are fine and arguably more idiomatic for
   the aesthetic. Pick one and commit. DO NOT MIX.

   If adopting Tailwind:
     - Install tailwindcss@4 (or latest stable), postcss, autoprefixer.
     - Configure tailwind.config.ts to read tokens from tokens.css via
       CSS variables. Extend theme.colors so that `text-fg` maps to
       `color: var(--fg)`, etc.
     - Disable default base for headings — keep them unstyled and let
       components control.
     - Keep globals.css minimal.

5. MDX wiring.
   Configure next.config.ts to parse .mdx files. Each MDX file imports
   a small set of components (H1, H2, P, Quote, DropCap, etc.) from a
   registry at src/components/mdx/. These components render MDX prose
   in the book-page style for Purposeless Efficiency, and in the
   dossier style elsewhere.

6. Env and config.
   - Create .env.example with documented vars (none required today;
     placeholder for future: RESEND_API_KEY, GITHUB_TOKEN, etc.).
   - Update next.config.ts: enable typedRoutes, strict TS, and pass
     through NEXT_PUBLIC_BUILD_VERSION from package.json version.
   - Update tsconfig: "strict": true (should already be true), add
     path aliases:
         "@/*": ["./src/*"]
         "@content/*": ["./src/content/*"]
         "@styles/*": ["./src/styles/*"]

7. CI.
   - .github/workflows/ci.yml already runs lint and build. Extend to:
     - Type check (tsc --noEmit).
     - Run scripts/lint-copy.ts from Prompt 13.
     - Playwright smoke test: load "/", verify the accordion renders,
       verify Cmd+K opens the palette.

8. Document.
   Update README.md:
     - State unambiguously that the Next.js app is canonical.
     - Point to docs/design-philosophy.md (Prompt 01) and
       docs/voice-guide.md (Prompt 13) as authoritative.
     - Describe the content layout: src/content/ is a CMS-in-git.

ACCEPTANCE CRITERIA

  - Legacy pages are either archived under public/_legacy/ or deleted.
  - public/js/ontology-data.js is copied into
    src/content/projects/theseus/ontology-data.raw.ts with a TODO
    note to convert to the Prompt 09 schema.
  - tools/ directory has been either archived or validated as still
    relevant.
  - Every dependency listed above is installed and its version is
    pinned with a caret range (^).
  - MDX compiles: a trivial test.mdx at src/content/test.mdx renders
    via a temporary /test route (route should be removed at end of

  - README.md reflects the new canonical stack and layout.

DO NOT
------
  - Do not introduce a second framework (Astro, SvelteKit, etc.).
    Next.js is the decision.
  - Do not set up Contentful, Sanity, or any headless CMS. The
    content layer is TypeScript modules and MDX files in the repo.
    This is deliberate — Michael is the only author.
  - Do not migrate the legacy /public site's functionality piece by
    piece into the Next app in this prompt. Migrate the DATA (like
    ontology-data.js), not the UI. UI is rebuilt from Prompt 02
    onward.
