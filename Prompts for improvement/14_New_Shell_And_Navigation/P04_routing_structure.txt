§B ACCEPTANCE
-------------
B1.  src/content/projects/index.ts exports:
       export const projects = [
         { slug: "hivemind", code: "HVM", title: "Hivemind",
           tagline: "<one line, ≤80 chars, drawn from existing thesis>",
           status: "in-progress" | "shipped" | "exploration" },
         { slug: "purposeless-efficiency", code: "PRP", title: "Purposeless Efficiency",
           tagline: "...", status: "..." },
         { slug: "theseus", code: "THS", title: "Theseus",
           tagline: "...", status: "..." },
       ];
     Tagline is sourced from existing repo content
     (src/content/projects/<slug>/thesis.ts, etc.) and capped at 80
     chars. If a tagline cannot be sourced from existing content,
     ask the implementer to flag and skip rather than invent.

B2.  /projects/page.tsx renders Container size="base" with:
       <h1>Projects</h1>
       Stack of three Card components, one per project. Each Card
       href={"/projects/" + slug}, title=title, meta={code + " · " + status},
       children=tagline.

B3.  Per-project routes /projects/[slug]/page.tsx — stubbed to
     render <StubPlaceholder /> for now. Suite 16/P04-P06 fills.

B4.  Old slug-direct routes /hivemind, /theseus,
     /purposeless-efficiency are removed and replaced by Next.js
     redirects in next.config.ts:
       redirects: async () => [
         { source: "/hivemind", destination: "/projects/hivemind", permanent: true },
         { source: "/purposeless-efficiency", destination: "/projects/purposeless-efficiency", permanent: true },
         { source: "/theseus", destination: "/projects/theseus", permanent: true },
       ]
     Permanent (308) so SEO carries over.

B5.  /chat/page.tsx is a stub that renders <StubPlaceholder /> for
     now. Suite 19 implements.

B6.  generateStaticParams in /projects/[slug]/page.tsx returns the
     three slugs from src/content/projects/index.ts. Unknown slugs
     trigger notFound().
